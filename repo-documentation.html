Timesheet App

app_timesheet — Full-stack employee time tracking application. React + Vite frontend, Node.js Express backend, SQLite in-memory database.

Architecture

Monorepo with two top-level directories:

  • backend/ — Express + SQLite + JWT + Joi + PDFKit + csv-writer
  • frontend/ — Vite + React + Material UI + React Query + Axios
AspectDetails
AuthJWT-based, 24h token expiry, email-only login
Data ModelUsers → Clients → Work Entries
SecurityRate limiting (5 attempts / 15 min), CORS, Helmet headers, parameterized queries
Frontend (Vite + React) --HTTP+JWT--> Backend (Express) --> SQLite (in-memory)

Capabilities

  • Client CRUD management
  • Work entry tracking with hours, dates, descriptions
  • Reports per client with CSV and PDF export
  • JWT authentication
  • Input validation with Joi schemas

Angular RealWorld

ts-angular-realworld-example-app — Angular 21 SPA implementing the RealWorld (Conduit / Medium clone) spec. Frontend-only, connects to external RealWorld API.

Architecture

  • Angular 21 with zoneless change detection
  • Build: Vite / AnalogJS
  • Feature-based structure:
    • core/ — services, interceptors, guards
    • features/ — auth, article, profile, editor
    • shared/ — pipes, directives
  • Auth: JWT in localStorage, injected via functional HTTP interceptor

Capabilities

  • User registration, login, profile management
  • Article CRUD with tags
  • Comments on articles
  • Follow users, favorite articles
  • Global and personal feeds
  • Settings / profile updates
  • Error handling for 400, 401, 403, 404, 500, network errors

How to Test

CommandDescription
npm run testUnit tests (Vitest)
npm run test:e2eE2E tests (Playwright)
npm run test:e2e:securitySecurity E2E tests
  • Playwright config: serial execution (workers: 1), Chromium only, 15s timeout per test
  • CI: GitHub Actions workflows for Playwright and security tests
  • E2E covers: auth flows, settings updates, error handling, article CRUD, network resilience

BDD Test Library

uc-bdd-test-generation-rest-api — Java library providing pre-built Cucumber/Gherkin step definitions for BDD testing of Spring Boot REST APIs. Zero-code BDD — write .feature files, get tests.

Architecture

  • Core: AbstractBddStepDefinition — base class with HTTP execution engine
  • Glue: DefaultRestApiBddStepDefinition — all Gherkin step bindings
  • Scope: ScenarioScope — stores variables across steps ($varName syntax)
  • Auth SPI: BddRestTemplateAuthentication interface for pluggable auth
  • WireMock integration for mocking third-party APIs
  • Uses TestRestTemplate for HTTP execution

Capabilities

  • Full HTTP verb support: GET, POST, PUT, PATCH, DELETE, HEAD
  • JSON body manipulation with JsonPath
  • Response validation: status codes, headers, body paths
  • Variable storage and reuse across scenarios
  • File-based request bodies
  • Query parameter support
  • Multipart / file upload steps
  • WireMock-based third-party API mocking
  • Pluggable authentication (JWT, Basic, custom)

How to Test

Maven dependency: fr.redfroggy.test.bdd:cucumber-restapi

  1. Create a CucumberTest runner class with @RunWith(Cucumber.class) and @CucumberOptions
  2. Create a @CucumberContextConfiguration class in the glue package
  3. Write .feature files under src/test/resources/features/
  4. Run: mvn test or via IDE

Example runner: RestApiCucumberTest.java with features in src/test/resources/features and glue fr.redfroggy.bdd.restapi.glue

CVE Remediation

uc-cve-remediation-regulatory-compliance — Spring Boot + MyBatis + Next.js full-stack app (RealWorld/Conduit spec) focused on CVE remediation and security compliance. Includes OWASP Dependency-Check and SonarQube integration.

Architecture

  • Backend: Spring Boot, MyBatis, SQLite, Flyway migrations, JWT auth
  • Frontend: Next.js with SWR
  • Dual API: REST controllers + Netflix DGS GraphQL
  • DDD four-layer: api → application → core → infrastructure
  • CQRS pattern for read/write separation
  • Security scanning: OWASP Dependency-Check (CVSS ≥ 7.0 fails build), SonarQube
  • Code formatting: Spotless

Capabilities

  • User registration, login, profile management
  • Article CRUD with tags, comments, favorites
  • User following
  • GraphQL API alongside REST
  • OWASP dependency CVE scanning
  • SonarQube static analysis
  • Seed data via Flyway (5 users, 4 articles, follows)
